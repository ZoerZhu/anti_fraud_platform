# 反诈骗学习平台详细优化任务清单

> 目标版本：校内试运行版  
> 优先方向：全模块体验完善，安全与可运行性作为准入项  
> 前端风格：白色主导、黑白简约、iOS 26 Liquid Glass、圆角、动态过渡

---

## 1. 优化目标

本阶段目标不是只做演示页，而是把项目整理到可在校内小范围试运行的状态。实施时需要同时满足三条底线：

1. 项目能从空库初始化、前后端能构建、主要流程能完整跑通。
2. 普通学生不能访问或执行管理员能力，上传、登录、CORS、敏感配置不留下明显风险。
3. 学生端学习体验、管理端内容维护、统计看板、AI 客服、推荐与积分成就形成完整闭环。

## 2. 当前已发现的重点问题

| 优先级 | 问题 | 影响 | 建议处理 |
| --- | --- | --- | --- |
| P0 | `sql/init.sql` 中 `sys_user.last_login_time` 重复创建 | 空库初始化可能失败 | 移除重复 `ALTER TABLE`，保留补丁脚本给旧库使用 |
| P0 | `backend/target` 被 Git 跟踪 | 构建会污染工作区，产物不应入库 | 从版本库移除构建产物，补充 `.gitignore` |

| P0 | 登录页“记住密码”保存明文密码到 `localStorage` | 明显安全风险 | 改为只记住用户名或取消该功能 |

| P1 | 统计服务存在 mock fallback 和口径不一致 | 后台数据可信度不足 | 统一真实统计口径，空数据明确显示 |
| P1 | 积分后台调分没有目标用户 ID | 管理端调分语义错误 | 调分 DTO 增加 `userId`，作用于目标学生 |

## 3. 阶段计划

### 阶段 0：工程基线整理

- 修复 SQL 初始化脚本，确保空库可一次性执行成功。
- 将增量脚本按用途命名并说明执行顺序，例如 `patch_user_last_login.sql` 仅给旧库使用。
- 从 Git 移除 `backend/target`，保留源码、配置样例和 SQL 脚本。
- 增加本地启动说明：MySQL、Redis、后端、前端、Swagger、默认账号。
- 明确 dev/prod 配置边界，敏感信息只通过环境变量或本地未提交配置注入。

验收标准：

- 新机器执行 `sql/init.sql` + seed 脚本后，后端能连接数据库。
- `git status` 不再因为构建产物产生大量变更。
- README 或本任务文档能指导开发者完成本地启动。

### 阶段 1：安全与权限准入

- 注册接口只允许创建学生账号，管理员账号由种子数据或后台维护。
- 登录失败提示收敛，避免区分“用户不存在”和“密码错误”造成枚举风险。
- 移除前端明文记住密码，最多保留记住用户名。
- 文件上传要求认证，限制图片 MIME、扩展名、大小、文件名、目录穿越。
- `/uploads/**` 保持匿名可读，方便图片展示。
- 用户管理、推荐规则、相似度计算、论坛置顶精选、积分调分、缓存状态等管理员接口补齐权限。
- CORS 按环境配置：开发允许本地端口，试运行只允许校内前端域名。
- JWT 黑名单检查失败时不阻塞登录态，但需要记录告警；退出登录 Redis 不可用时返回明确降级信息。

验收标准：

- 学生账号无法访问管理端接口。
- 未登录用户无法上传文件。
- 登录页不再存储密码。
- 安全相关接口返回 401/403 语义清晰。

### 阶段 2：前端类型与请求层统一

- 将 `frontend/src/utils/request.ts` 调整为统一返回业务数据 `T`。
- 所有 API 文件从 `AxiosPromise<T>` 改为 `Promise<T>`。
- 页面不再混用 `res.data` 与直接业务数据。
- 统一 `PageResult<T>`、`UserVO`、`NewsVO`、`CaseVO`、`ChallengeVO`、`PostVO`、`DashboardVO` 类型。
- 修复 Cropper 类型、图标模块声明、未使用变量、nullable 数据、管理员关卡编辑器类型错误。
- 保留严格 TypeScript 配置，不通过关闭 `strict` 或 `noUnused*` 绕过问题。

验收标准：

- `npm run build` 通过。
- 所有页面接口调用方式一致。
- 请求错误在页面侧表现统一，不出现重复弹窗或空白页。

## 4. 前端视觉与交互优化

### 4.1 总体设计方向

项目视觉应从当前普通后台/内容站风格升级为“轻量、干净、可信、现代”的校园安全学习产品：

- 主背景以白色、近白色、浅灰为主。
- 文本、图标、导航以黑、深灰、炭黑为主。
- 允许少量功能性色彩：风险红、提醒橙、安全绿、信息蓝，但只作为状态与重点，不主导整体画面。
- 不使用大面积彩色渐变、紫蓝渐变、深色科技风、厚重阴影或过度装饰背景。
- 学习端比管理端更有亲和力；管理端保持高密度、清晰、可扫描。

建议基础色板：

| Token | 用途 | 建议值 |
| --- | --- | --- |
| `--color-bg` | 页面背景 | `#f7f8fa` |
| `--color-surface` | 主内容面 | `#ffffff` |
| `--color-surface-soft` | 次级面 | `#f2f4f7` |
| `--color-text` | 主文本 | `#111318` |
| `--color-text-muted` | 次级文本 | `#667085` |
| `--color-border` | 边线 | `rgba(17, 19, 24, 0.08)` |
| `--color-danger` | 高风险/错误 | `#d92d20` |
| `--color-warning` | 预警 | `#dc6803` |
| `--color-success` | 安全/通过 | `#039855` |
| `--color-info` | 信息/推荐 | `#1570ef` |

### 4.2 iOS 26 Liquid Glass 使用原则

液态玻璃只用于关键前景控件，不把所有卡片都做成玻璃。

适合使用液态玻璃的区域：

- 顶部导航栏、侧边栏当前项、移动端底部导航。
- 首页搜索框、筛选栏、推荐类型切换。
- 浮动操作按钮、返回顶部、快捷发布按钮。
- 弹窗、抽屉、确认面板、图片预览层。
- 管理端工具栏、表格筛选条、批量操作条。
- 闯关结果面板、FSM 情景模拟选择面板。

不建议使用液态玻璃的区域：

- 大段正文、表格主体、富文本内容区。
- 密集数据图表卡片主体。
- 评论内容列表主体。
- 表单里每一个输入框外层。

### 4.3 液态玻璃组件规范

新增全局工具类建议放在 `frontend/src/styles/index.scss` 或拆分为 `frontend/src/styles/tokens.scss` 与 `frontend/src/styles/liquid-glass.scss`。

核心要求：

- 使用 `backdrop-filter` 与 `-webkit-backdrop-filter`。
- 使用半透明白色底、细边框、内发光、弱阴影。
- 使用伪元素做高光和内折射，不额外增加无意义 DOM。
- 提供 `@supports not` fallback，浏览器不支持模糊时显示为高不透明白色。
- 支持 `prefers-reduced-motion: reduce`，关闭浮动、闪光、形变动画。
- 文本对比度优先于玻璃效果，正文不因透明而变浅。

建议类名：

- `.liquid-glass`：基础玻璃材质。
- `.liquid-pill`：胶囊按钮、筛选项、导航项。
- `.liquid-panel`：弹窗、抽屉、工具栏。
- `.liquid-toolbar`：管理端筛选与批量操作区。
- `.liquid-floating`：悬浮操作入口。

### 4.4 圆角规范

| 场景 | 圆角 |
| --- | --- |
| 图标按钮、胶囊导航、筛选项 | `999px` |
| 普通按钮、输入框、选择器 | `12px` |
| 小卡片、列表项 | `14px` |
| 弹窗、抽屉、重点面板 | `18px - 24px` |
| 表格、数据面板 | `10px - 14px` |

注意：

- 不要所有东西都巨大圆角。
- 管理端表格和密集信息面板圆角要更克制。
- 卡片不要套卡片，避免视觉层级混乱。

### 4.5 动态过渡规范

统一过渡参数：

- 普通 hover：`160ms - 220ms ease-out`
- 弹窗/抽屉：`220ms - 320ms cubic-bezier(0.2, 0.8, 0.2, 1)`
- 页面局部切换：`180ms - 260ms`
- 按压反馈：轻微 `translateY(1px)` 或 `scale(0.985)`

交互状态：

- 按钮 hover：边框略亮、背景透明度略升、阴影轻微增强。
- 按钮 active：轻微下压，不改变布局尺寸。
- 输入框 focus：黑色或信息蓝焦点环，保持高对比。
- 卡片 hover：仅列表项和可点击卡片有轻微浮起，静态信息卡不动画。
- 列表加载：使用 skeleton 或淡入，不使用大面积旋转动画。

无障碍要求：

- 支持键盘 focus ring。
- 支持 reduced motion。
- 文本不覆盖图标、不溢出按钮。
- 移动端 360px 宽度下按钮文字不挤压。

### 4.6 页面级优化任务

#### 登录与注册

- 改成白色极简布局，登录卡片使用轻玻璃面板。
- 左侧或顶部加入平台名称、反诈学习定位和安全感视觉，但不要做营销落地页。
- 删除明文记住密码，改为记住用户名。
- 表单错误提示更具体，登录中禁用按钮并显示 loading。
- 回车提交、防重复提交、跳转 redirect 保持稳定。

#### 首页

- 首页作为真实工作台，不做空洞宣传页。
- 首屏展示：必读预警、继续学习、推荐案例、下一关卡、积分/成就摘要。
- 使用白色背景和浅灰分区，顶部工具条可用液态玻璃。
- 推荐卡片展示原因标签，如“必读”“与你的弱点相关”“热门案例”。

#### 资讯中心

- 筛选栏使用胶囊液态玻璃控件。
- 资讯卡片以清晰标题、摘要、分类、必读/置顶标签为主。
- 详情页正文保持纯白高可读，不使用透明背景。
- 浏览、点赞、返回列表、相关推荐交互统一。

#### 案例展示

- 案例列表突出风险等级、诈骗类型、标签和热度。
- 风险色只用于等级徽标，不铺满页面。
- 案例详情增加“识别要点”“防范建议”“相似案例”结构。
- 交互式剧本或决策树内容需要稳定布局，避免节点跳动。

#### 知识闯关与情景模拟

- 关卡列表展示锁定状态、通关状态、最高分、奖励积分。
- 题目页减少干扰，选项按钮具备清晰选中、正确、错误状态。
- 结果页用液态玻璃面板展示分数、评级、奖励和复盘入口。
- FSM 情景模拟的选择按钮使用圆角、按压反馈和风险提示动效。

#### 论坛社区

- 发帖入口可使用悬浮液态按钮。
- 帖子列表保持白底高可读，置顶/精选用轻标签区分。
- 评论树层级清晰，回复、点赞、删除按钮使用图标+轻文字。
- 管理端审核操作需要二次确认。

#### 智能客服

- 聊天窗口以白色为主，消息气泡黑白简约。
- 用户消息和 AI 消息用轻微灰阶区分。
- 输入栏可使用液态玻璃浮层。
- AI 不可用时显示明确降级提示，不暴露后端异常细节。

#### 个人中心、积分、成就

- 个人信息、学习记录、积分等级、成就墙统一视觉。
- 成就卡片使用低饱和图标和状态标签，避免彩色堆叠。
- 弱点标签、兴趣标签作为可解释推荐入口。
- 头像上传流程补齐裁剪、预览、失败回退。

#### 管理端

- 管理端整体保持白色、高密度、表格优先。
- 侧边栏和顶部栏可用轻玻璃，但表格主体保持纯白。
- 筛选栏、批量操作条、弹窗使用液态玻璃面板。
- 管理端按钮优先使用图标与清晰动词，不堆叠过多彩色按钮。
- 看板图表保持浅色背景，图表色不超过 4 种主状态色。

## 5. 后端模块优化任务

### 用户与认证

- 注册 DTO 增加用户名、密码、手机号、邮箱、学号格式校验。
- 登录接口统一失败提示，登录成功更新最后登录时间。
- 用户信息接口统一返回脱敏数据，不返回密码字段。
- 用户列表、用户详情、更新、启禁用均限定管理员。
- 禁用用户后拒绝登录，必要时使现有 token 失效。

### 资讯模块

- 创建、更新资讯使用 `@Valid`。
- 资讯详情增加浏览记录与浏览量更新的幂等策略。
- 点赞、取消点赞处理重复请求并返回稳定结果。
- 必读、置顶、发布状态变化后刷新缓存。
- 浏览历史返回包含浏览时间和停留时长的 VO，而不是裸 `News`。

### 案例模块

- 案例创建/更新校验标题、内容、类型、风险分、难度、标签。
- 标签删除前检查关联案例，或明确级联策略。
- 浏览记录写入用户画像和学习活动。
- 热门案例和 Wilson 排序结果刷新缓存。
- 详情页返回是否点赞、标签、推荐相似案例所需字段。

### 闯关与情景模拟

- 关卡保存时校验 quiz content 或 scenario scripts，避免脏 JSON。
- FSM 校验起始节点、结束节点、边引用节点、重复 edge id。
- 提交答案时校验题号、选项索引、空答案、重复提交。
- 情景模拟通关记录与普通闯关统计统一。
- 奖励积分和成就解锁保持事务一致或具备补偿日志。

### 论坛与评论

- 发帖、更新、评论增加内容长度和空白校验。
- 帖子置顶、精选仅管理员可用。
- 删除帖子时处理点赞、评论、评论点赞级联。
- 评论删除后正确维护评论数。
- 普通用户只能改删自己的内容，管理员可治理全部内容。

### AI 客服

- 问题不能为空，限制最大长度。
- 历史消息按 user/assistant 成对传递，不只传 assistant。
- API key 缺失时返回可理解提示。
- DeepSeek 请求失败不泄露完整异常给前端。
- 增加 token 使用统计、反馈统计和会话归属校验。

### 推荐模块

- 推荐规则增删改查仅管理员可用。
- 相似度批量计算仅管理员或定时任务触发。
- 推荐曝光记录和点击反馈保持一致，避免推荐一次后永久排除。
- 新用户冷启动优先必读资讯、热门案例、基础闯关。
- 推荐理由返回给前端用于解释，而不是只返回分数。

### 积分与成就

- 后台调分 DTO 增加 `userId`。
- 积分增加、扣减、等级更新记录操作原因。
- 成就解锁保持幂等，避免并发重复加分。
- 连续学习统计统一来源：资讯浏览、案例浏览、闯关、发帖评论。
- 个人中心提供积分、等级、成就、学习历史聚合接口。

### 统计与缓存

- 移除随机 mock 作为正式数据源，空数据返回空数组和默认 0。
- 统一日活、新增用户、浏览量、闯关完成率、院系平均分口径。
- 导出接口支持日期范围参数并校验。
- Redis `keys` 模糊删除仅用于小规模开发，试运行可改为命名集合或 scan。
- 缓存状态接口返回真实 key 命中、TTL、刷新时间。

## 6. 数据与脚本优化

- `init.sql` 只负责新库完整初始化，不混入已建库补丁。
- `patch_*.sql` 只负责老版本升级，脚本顶部注明适用版本。
- 种子数据分模块导入：资讯、案例、关卡、论坛、推荐规则。
- 默认管理员账号必须在文档中标注仅供开发/试运行使用，首次部署后要求修改密码。
- 演示数据应覆盖：
  - 学生账号、管理员账号。
  - 多个资讯分类和必读资讯。
  - 多种案例标签和风险等级。
  - quiz 关卡与 scenario 关卡。
  - 初始成就、推荐规则、统计样例。

## 7. 测试与验收清单

### 构建检查

- 后端：`mvn -DskipTests compile`
- 前端：`npm run build`
- SQL：空库执行 `init.sql`，再执行 seed 脚本。

### 核心联调流程

1. 学生注册并登录。
2. 浏览必读资讯，点赞资讯。
3. 浏览案例，点赞案例，产生学习记录。
4. 完成一个 quiz 闯关。
5. 完成一个 scenario 情景模拟。
6. 查看积分、等级、成就变化。
7. 查看推荐页，点击推荐内容。
8. 发帖、评论、点赞。
9. 使用 AI 客服新建会话、提问、反馈。
10. 管理员登录，维护资讯、案例、关卡、论坛内容。
11. 管理员查看统计看板并导出 Excel。

### UI 验收

- 页面以白色和浅灰为主，黑色文字高对比。
- 不出现大面积彩色背景或厚重渐变。
- 液态玻璃只用于导航、工具栏、浮层、弹窗、筛选、关键操作。
- hover、active、focus、disabled 状态完整。
- 移动端 360px 宽度下内容不重叠、不溢出。
- `prefers-reduced-motion` 下关闭明显动画。
- 浏览器不支持 `backdrop-filter` 时仍然可读可用。

### 安全验收

- 未登录无法访问学生功能接口。
- 学生无法访问管理端接口。
- 未登录无法上传文件。
- 前端不保存明文密码。
- 上传文件不能覆盖任意路径。
- 生产配置不包含真实密钥和默认弱口令。
